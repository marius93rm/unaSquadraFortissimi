# Planning Contract

Use this skill when a task is too large or ambiguous to execute safely in one pass.

## Workflow

1. Ground the plan in repository evidence.
2. State the goal, success criteria, constraints, and non-goals.
3. Choose a minimal implementation approach that matches existing patterns.
4. Define public interface changes, data flow, and edge cases.
5. Define verification steps and acceptance criteria.
6. Call out assumptions explicitly.

## Plan Shape

Keep plans concise and executable:

- summary
- implementation changes
- interface or behavior changes
- test plan
- assumptions and risks

Do not invent architecture, schema, or rollout policy beyond what is needed to avoid implementation mistakes.
