# Release Notes

Use this skill to translate implementation into useful release communication.

## Workflow

1. Inspect commits, diffs, PR notes, or issue references.
2. Separate user-visible changes from internal maintenance.
3. Group by added, changed, fixed, deprecated, removed, security, and known issues as relevant.
4. Mention migration or action required.
5. Keep language factual and concise.

## Output

```markdown
## Added
## Changed
## Fixed
## Security
## Migration Notes
## Known Issues
```

Omit empty sections unless the project convention requires them.
