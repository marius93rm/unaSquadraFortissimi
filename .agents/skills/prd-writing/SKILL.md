# PRD Writing

Use this skill to write requirements that engineering can execute and verify.

## Workflow

1. Define the user problem and target audience.
2. State goals, non-goals, assumptions, and constraints.
3. Describe user journeys and core requirements.
4. Convert requirements into testable acceptance criteria.
5. Include metrics, rollout notes, dependencies, and open questions.
6. Keep the PRD concise; link details instead of embedding everything.

## PRD Shape

```markdown
# PRD: <Feature>
## Problem
## Goals and Non-Goals
## Users and Jobs
## Requirements
## Acceptance Criteria
## Metrics
## Rollout and Dependencies
## Open Questions
```

Avoid letting customers prescribe implementation. Capture the problem behind each requested feature.
