# Technical Writing

Use this skill to make repo knowledge accurate and maintainable.

## Workflow

1. Identify the reader and task.
2. Verify commands, paths, options, and behavior from the repo.
3. Put the fastest successful path first.
4. Preserve gotchas, prerequisites, and failure modes.
5. Remove stale or duplicate instructions.
6. Keep examples copy-pasteable and scoped.

## Checks

- Paths exist.
- Commands match package scripts or documented tooling.
- Environment variables are named but secrets are never included.
- Claims are either verified or clearly marked as assumptions.
