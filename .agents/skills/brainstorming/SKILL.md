# Brainstorming

Use this skill to turn an unclear idea into a buildable direction before implementation.

## Workflow

1. Inspect the current repo surface and any relevant docs.
2. Restate the user goal, constraints, and known unknowns.
3. Ask only the questions that materially change the design. Prefer one question at a time.
4. Offer 2-3 approaches when tradeoffs are real.
5. Recommend one approach and state what is intentionally out of scope.
6. Convert the decision into acceptance criteria or hand off to `writing-plans`.

## Guardrails

- Do not use this for tiny mechanical edits.
- Do not block progress on questions that local evidence or a conservative assumption can answer.
- Do not turn brainstorming into a speculative roadmap.
- If the work is too large, split it into independently shippable pieces.

## Output

- problem statement
- recommended approach
- rejected alternatives and why
- success criteria
- open decisions that truly need the user
