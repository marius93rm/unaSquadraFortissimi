# Systematic Debugging

Use this skill to avoid guess-and-patch debugging.

## Workflow

1. Capture the exact symptom, command, input, log, stack trace, or reproduction.
2. Shrink the failure to the smallest observable case.
3. Map the path from trigger to symptom.
4. Form ranked hypotheses with evidence and disconfirming checks.
5. Test the highest-value hypothesis.
6. Fix the cause, not just the symptom.
7. Verify normal path, failure path, and recurrence risk.

## Output

Report:

- root cause or strongest remaining hypothesis
- evidence supporting it
- fix strategy
- validation performed
- unresolved uncertainty
